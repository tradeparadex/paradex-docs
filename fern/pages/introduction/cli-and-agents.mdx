## Fetch the Full Documentation

Clone the complete Paradex docs repo for local reference, AI agent context, or offline use:

```bash
git clone https://github.com/tradeparadex/paradex-docs.git
```

To update to the latest version:

```bash
git -C paradex-docs pull origin main
```

<Note>
The docs source lives in `fern/pages/` as `.mdx` files. The REST API spec is at `fern/apis/prod_rest/openapi/openapi.yml` (OpenAPI 3.0) and the WebSocket spec is at `fern/apis/prod_ws/asyncapi/asyncapi.yml` (AsyncAPI 2.6).
</Note>

### Feeding Docs to an AI Agent

If you're using an AI coding assistant (Claude Code, Cursor, Copilot, etc.), point it at the cloned repo for full context:

```bash
# Clone once
git clone https://github.com/tradeparadex/paradex-docs.git

# In your project's AI config (e.g. CLAUDE.md), add:
# Local path: ./paradex-docs
# Use these docs as reference when answering questions about Paradex features, endpoints, or behavior
```

For a quick API-only reference without cloning the full repo, you can fetch the OpenAPI spec directly:

```bash
# REST API spec (OpenAPI 3.0)
curl -sL https://raw.githubusercontent.com/tradeparadex/paradex-docs/main/fern/apis/prod_rest/openapi/openapi.yml -o paradex-rest-api.yml

# WebSocket API spec (AsyncAPI 2.6)
curl -sL https://raw.githubusercontent.com/tradeparadex/paradex-docs/main/fern/apis/prod_ws/asyncapi/asyncapi.yml -o paradex-ws-api.yml
```

## API Endpoints

| Environment | REST API | WebSocket |
|:------------|:---------|:----------|
| **Production** | `https://api.prod.paradex.trade/v1` | `wss://ws.api.prod.paradex.trade/v1` |
| **Testnet** | `https://api.testnet.paradex.trade/v1` | `wss://ws.api.testnet.paradex.trade/v1` |

Test connectivity from your terminal:

```bash
# List all available markets
curl -s https://api.prod.paradex.trade/v1/markets | python -m json.tool | head -50

# Get system config
curl -s https://api.prod.paradex.trade/v1/system/config | python -m json.tool

# Get BBO for BTC
curl -s "https://api.prod.paradex.trade/v1/bbo?market=BTC-USD-PERP" | python -m json.tool
```
