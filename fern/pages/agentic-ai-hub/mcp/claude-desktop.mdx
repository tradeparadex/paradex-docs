# Claude Desktop

<Tabs>
<Tab title="Local MCP setup">

Full API access including trading. Requires installing a few tools on your machine.

<Steps>
### Install prerequisites

Most steps require running commands in a terminal:

- **macOS:** Open Terminal — press **Cmd + Space**, type "Terminal", and press Enter.
- **Windows:** Right-click the Windows Start icon and select **Terminal**.

#### Install Python

Python 3.10 or higher is required. Check your version:

```bash
python3 --version
```

If the version is below 3.10 or the command is not found, install Python from [python.org/downloads](https://www.python.org/downloads/).

#### Install uvx

`uvx` is a fast Python package runner used to download and start the MCP server automatically.

<Tabs>
<Tab title="pip">
```bash
pip install uv
```
</Tab>
<Tab title="Homebrew (macOS)">
```bash
brew install uv
```
</Tab>
</Tabs>

Verify the installation:

```bash
uvx --version
```

<Accordion title="Alternative: Run with Docker">

If you prefer not to install Python or uvx directly, you can run the MCP server inside a Docker container. Requires Docker.

```bash
git clone https://github.com/tradeparadex/mcp-paradex-py.git
cd mcp-paradex-py
docker build . -t mcp-paradex-py

# Read-only mode (public market data only)
docker run --rm -i mcp-paradex-py

# With trading capabilities
docker run --rm -e PARADEX_ACCOUNT_PRIVATE_KEY=your_key -i mcp-paradex-py
```

</Accordion>

### Get your Paradex private key

All trading features require a Paradex account private key. Market data tools work without one — you can skip this step if you only need market data.

1. Go to [Paradex](https://app.paradex.trade) and connect your wallet.
2. Navigate to **Settings → API Keys** and create a new key.
3. Copy the private key — you'll use it below.

<Warning>
Your private key controls access to your trading account. Never share it publicly or commit it to version control. For more details, see [Authentication](/api/general-information/authentication).
</Warning>

<Note>
**Testnet first?** Use `PARADEX_ENVIRONMENT=testnet` and get test funds from the Paradex testnet faucet.
</Note>

### Configure Claude Desktop

1. Open Claude Desktop → **Settings → Developer → Edit Config**. This opens the config file path directly. Find and open `claude_desktop_config.json` with your text editor.

   If the config file doesn't exist yet, create it at:
   - **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
   - **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

2. Paste the following into the config file. Replace `your_private_key_here` with your actual key:

   ```json
   {
     "mcpServers": {
       "paradex": {
         "command": "uvx",
         "args": ["mcp-paradex"],
         "env": {
           "PARADEX_ENVIRONMENT": "prod",
           "PARADEX_ACCOUNT_PRIVATE_KEY": "your_private_key_here"
         }
       }
     }
   }
   ```

3. Restart Claude Desktop. It will automatically download and start the MCP server via `uvx`.

### Verify it's working

Click the **+** icon in the bottom-left of the chat input → **Connectors**. You should see **mcp paradex** listed with its toggle enabled.

<img src="../../../assets/agentic-ai-hub/claude-desktop-connectors.png" alt="Claude Desktop Connectors panel showing mcp paradex enabled" style={{width:"100%",display:"block",marginTop:"0.75rem"}} />

Then ask Claude:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're all set. To test trading access:

> "Show me my Paradex account summary."

</Steps>

### Troubleshooting

**Tools not appearing / MCP not loading**
- Make sure `uvx` is installed: `uvx --version`
- Check that the config file has valid JSON (no trailing commas)
- Restart Claude Desktop after config changes

**Authentication errors**
- Double-check your private key is correct and starts with `0x`
- Verify `PARADEX_ENVIRONMENT` matches where your account was created (`prod` or `testnet`)

**`uvx: command not found`**
```bash
pip install uv
# or on macOS:
brew install uv
```

**Want to test without a private key first?** Market data tools work without any credentials. Just omit `PARADEX_ACCOUNT_PRIVATE_KEY` from the config.

<Steps>
### Add the MCP server

1. Open Claude Desktop → **Settings → Integrations** (or **Settings → Connectors**).
2. Add a new MCP server.
3. Enter `https://mcp.paradex.trade/mcp`.

### Verify it's working

Ask Claude:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're connected.
</Steps>

<Steps>
### Connect through Smithery

1. Visit [smithery.ai](https://smithery.ai) and search for **Paradex**.
2. Select the Paradex MCP server.
3. Choose **Claude Desktop** as your client.
4. Configure your settings and click **Connect**.

### Verify it's working

Ask Claude:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're connected.
</Steps>

<Steps>
### Install via Smithery CLI

Run this in your terminal:

```bash
npx -y @smithery/cli@latest mcp add @tradeparadex/mcp-paradex-py --client claude
```

You will be required to create an account or sign in to Smithery. Once signed in, a browser window opens with configuration settings — no fields are required, leave `paradexAccountPrivateKey` empty (trading is not supported in this mode). Click **Connect**.
