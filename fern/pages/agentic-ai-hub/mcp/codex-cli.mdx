### Step 3 — Configure

<Note>
This step uses the OpenAI Codex CLI. If you haven't installed it yet, run `npm install -g @openai/codex` or see [Codex CLI on GitHub](https://github.com/openai/codex) for details.
</Note>

Run this command in your terminal:

```bash
codex mcp add paradex -- uvx mcp-paradex
```

Or edit `~/.codex/config.toml` in a text editor:

```toml
[mcp_servers.paradex]
command = "uvx"
args = ["mcp-paradex"]
env = { PARADEX_ENVIRONMENT = "prod", PARADEX_ACCOUNT_PRIVATE_KEY = "your_private_key_here" }
```

Replace `your_private_key_here` with your actual key from Step 2.

The config is shared between the Codex CLI and the Codex VS Code extension — configure once and both clients pick it up.

---

### Verify It's Working

Ask Codex:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're all set. To test trading access:

> "Show me my Paradex account summary."

---

### Troubleshooting

**Tools not appearing / MCP not loading**
- Make sure `uvx` is installed: `uvx --version`
- Re-run the `codex mcp add` command or check `~/.codex/config.toml`

**`uvx: command not found`**
```bash
pip install uv
# or on macOS:
brew install uv
```

## B. Paradex-hosted MCP

<Warning>
This setup does **not support trading** at the moment. Only public market data is available.
</Warning>

### Via Smithery

1. Visit [smithery.ai](https://smithery.ai) and search for **Paradex**.
2. Select the Paradex MCP server.
3. Choose **OpenAI Codex CLI** as your client.
4. Follow the guided setup and click **Connect**.

### Verify It's Working

Ask Codex:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're connected. Note that trading features are not available with this setup.
