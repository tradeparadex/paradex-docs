# CLI

**Repository:** [github.com/tradeparadex/paradex-cli](https://github.com/tradeparadex/paradex-cli)

A crypto DEX AI-native command-line interface for managing Paradex account contracts. Designed for both direct use and AI agent integration, the CLI provides full Paradex API access in a single binary.

What the CLI supports:
- Fetch live and historical market data
- Manage trading accounts and vaults
- Place, monitor, and cancel orders
- Track positions, balances, and funding payments

<Warning>
Gigavault operations are not currently supported through the CLI. The CLI focuses on account contract management — guardians, signers, transaction signing, and emergency operations.
</Warning>

## Commands

All commands accept an `--env` flag to specify the target environment (`prod`, `testnet`, or `nightly`).

### Account Management

```bash
# Print account information
paradex_cli print-account-info ACCOUNT_ADDRESS --env ENVIRONMENT

# Add a guardian
paradex_cli add-guardian PUB_KEY --env ENVIRONMENT

# Add a backup guardian
paradex_cli add-guardian-backup PUB_KEY --env ENVIRONMENT

# Change the account signer
paradex_cli change-signer PUB_KEY --env ENVIRONMENT
```

### Transaction Signing & Submission

```bash
# Sign an invoke transaction
paradex_cli sign-invoke-tx FILE_PATH --env ENVIRONMENT

# Submit a signed invoke transaction
paradex_cli submit-invoke-tx TX_FILE_PATH SIG_FILES --env ENVIRONMENT
```

### Emergency Operations

```bash
# Trigger guardian escape mechanism
paradex_cli trigger-escape-guardian --env ENVIRONMENT
```

## AI Agent Integration

AI assistants like Claude, Cursor, Codex, Copilot, Gemini, Goose, and OpenClaw can use CLI directly to interact with Paradex APIs — no separate server needed.

### CLI vs. Standalone MCP Server

Paradex offers two ways to connect an AI agent:

| | CLI (`paradex_cli`) | Standalone MCP Server (`mcp-paradex`) |
|---|---|---|
| **Install** | `pip install paradex_cli` | Auto-downloaded via `uvx` — no install step |
| **Focus** | Account & contract management (guardians, signers, transaction signing) + trading | Trading, market data, and portfolio analysis |
| **MCP tools** | CLI commands exposed as MCP tools | 25+ purpose-built trading tools |
| **Best for** | Managing your Paradex account through an AI agent | Connecting an AI assistant to live markets for trading and analysis |

**When to use the CLI:** You need AI-assisted account operations — adding guardians, changing signers, signing transactions — alongside trading.

**When to use the standalone MCP server:** You want your AI assistant focused on trading — market data, order management, positions, vaults, and funding analysis. This is the right choice for most users.

You can also use both side by side. They register as separate MCP servers and don't conflict.

### Setup

Install the CLI:

```bash
pip install paradex_cli
```

Then add it as an MCP server in your AI client. The setup is the same as the [standalone MCP server](/agentic-ai-hub/mcp/quickstart) — just replace the command.

<CodeBlocks>
<CodeBlock title="Claude Code">
```bash
claude mcp add paradex-cli paradex_cli mcp \
  -e PARADEX_ENVIRONMENT=prod \
  -e PARADEX_ACCOUNT_PRIVATE_KEY=your_private_key_here
```
</CodeBlock>
<CodeBlock title="Claude Desktop / Cursor / Windsurf">
```json
{
  "paradex-cli": {
    "command": "paradex_cli",
    "args": ["mcp"],
    "env": {
      "PARADEX_ENVIRONMENT": "prod",
      "PARADEX_ACCOUNT_PRIVATE_KEY": "your_private_key_here"
    }
  }
}
```
</CodeBlock>
</CodeBlocks>

See the [Quickstart Guide](/agentic-ai-hub/mcp/quickstart) for where to put this config in each client.
