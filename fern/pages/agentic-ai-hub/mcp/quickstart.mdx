# Quickstart guide

This guide helps you connect the Paradex MCP server to your AI assistant in minutes.

Your setup choice depends on two things:

- **Which MCP features you need** — trading is only available with a local setup at the moment. Remote and Paradex-hosted options provide market data only.
- **Which AI client you prefer** — Claude Desktop, Cursor, ChatGPT, and others are supported. Not every client supports every setup path.

### Pick your setup path

<Tabs>
<Tab title="Local MCP setup (recommended)">

MCP server runs on your machine.

- Full API support including trading
- Requires prerequisites (Python 3.10+, uvx) and a Paradex private key
- Steps are included on each client page

Click your client name in the table above to go to its setup page — each page includes prerequisites, private key setup, and configuration instructions.

</Tab>
<Tab title="Remote MCP deployment">

You host your own MCP server on a cloud platform (Railway, Render, Fly.io, etc.) and connect your AI client to it via a URL.

- No trading support at the moment — market data only
- Required for clients that cannot run local programs (such as Claude.ai web)
- Also available for ChatGPT Desktop as an alternative to local setup

Follow the [Remote Deployment guide](/agentic-ai-hub/mcp/remote-deployment) to deploy your server, then return to your client's page for instructions on how to connect it.

</Tab>
<Tab title="Paradex-hosted MCP">

The fastest way to get started. You connect your AI client directly to the Paradex-hosted MCP server — no installation, no cloud deployment.

- No trading support at the moment — market data only
- Just add a URL to your client or use Smithery for guided setup

Each client page includes a "Paradex-hosted MCP" tab with instructions. You can also see the [dedicated guide](/agentic-ai-hub/mcp/connect-to-paradex-mcp) for all options in one place.

</Tab>
</Tabs>
