# Using CLI with AI Agents

The Paradex CLI includes a built-in MCP server, so AI assistants like Claude, Cursor, Codex, Copilot, Gemini, Goose, and OpenClaw can use it directly — no separate server needed.

## CLI vs. Standalone MCP Server

Paradex offers two ways to connect an AI agent:

| | CLI (`paradex_cli`) | Standalone MCP Server (`mcp-paradex`) |
|---|---|---|
| **Install** | `pip install paradex_cli` | Auto-downloaded via `uvx` — no install step |
| **Focus** | Account & contract management (guardians, signers, transaction signing) + trading | Trading, market data, and portfolio analysis |
| **MCP tools** | CLI commands exposed as MCP tools | 25+ purpose-built trading tools |
| **Best for** | Managing your Paradex account through an AI agent | Connecting an AI assistant to live markets for trading and analysis |

**When to use the CLI:** You need AI-assisted account operations — adding guardians, changing signers, signing transactions — alongside trading.

**When to use the standalone MCP server:** You want your AI assistant focused on trading — market data, order management, positions, vaults, and funding analysis. This is the right choice for most users.

You can also use both side by side. They register as separate MCP servers and don't conflict.

## Setup

Install the CLI:

```bash
pip install paradex_cli
```

Then add it as an MCP server in your AI client. The setup is the same as the [standalone MCP server](/agentic-ai-hub/mcp/installation) — just replace the command.

<CodeBlocks>
<CodeBlock title="Claude Code">
```bash
claude mcp add paradex-cli paradex_cli mcp \
  -e PARADEX_ENVIRONMENT=prod \
  -e PARADEX_ACCOUNT_PRIVATE_KEY=your_private_key_here
```
</CodeBlock>
<CodeBlock title="Claude Desktop / Cursor / Windsurf">
```json
{
  "paradex-cli": {
    "command": "paradex_cli",
    "args": ["mcp"],
    "env": {
      "PARADEX_ENVIRONMENT": "prod",
      "PARADEX_ACCOUNT_PRIVATE_KEY": "your_private_key_here"
    }
  }
}
```
</CodeBlock>
</CodeBlocks>

See the [Quickstart Guide](/agentic-ai-hub/mcp/installation) for where to put this config in each client.
