# Claude.ai (Web)

Claude.ai web cannot run local programs. This page covers two setup options — choose one:

- [**Remote MCP Deployment**](#remote-mcp-deployment) — Market data only. Host your own MCP server on a cloud platform.
- [**Paradex-hosted MCP**](#paradex-hosted-mcp) — Market data only. No installation needed, just add a URL.

## Remote MCP Deployment

<Warning>
Remote setups do **not support trading** at the moment — only market data is available.
</Warning>

First, deploy the MCP server to a cloud platform by following the [Remote Deployment guide](/agentic-ai-hub/mcp/remote-deployment). Once you have your endpoint URL, return here to connect it to Claude.ai.

<Steps>
  <Step title="Open Settings">
    Open [claude.ai](https://claude.ai) → **Settings → Integrations**.
  </Step>
  <Step title="Add MCP Server">
    Click **Add MCP Server**.
  </Step>
  <Step title="Enter your endpoint URL">
    Enter your endpoint URL (e.g. `https://your-domain.example.com/mcp`).
  </Step>
</Steps>

## Paradex-hosted MCP

<Warning>
This setup does **not support trading** at the moment. Only public market data is available.
</Warning>

### Via URL

<Steps>
  <Step title="Open Settings">
    Open [claude.ai](https://claude.ai) → **Settings → Integrations**.
  </Step>
  <Step title="Add MCP Server">
    Click **Add MCP Server**.
  </Step>
  <Step title="Enter the Paradex MCP URL">
    Enter `https://mcp.paradex.trade/mcp`.
  </Step>
</Steps>

### Via Smithery

<Steps>
  <Step title="Find the Paradex server">
    Visit [smithery.ai](https://smithery.ai) and search for **Paradex**.
  </Step>
  <Step title="Select and configure">
    Select the Paradex MCP server, choose **Claude** as your client.
  </Step>
  <Step title="Connect">
    Follow the guided setup and click **Connect**.
  </Step>
</Steps>

### Verify It's Working

Ask Claude:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're connected.

### Troubleshooting

**MCP server not loading**
- Check that the URL is correct and ends with `/mcp`
- Try removing and re-adding the server in **Settings → Integrations**
- For remote deployments, verify your server is running: `curl https://your-endpoint.example.com/health`
