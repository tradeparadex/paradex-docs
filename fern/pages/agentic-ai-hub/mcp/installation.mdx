# Quickstart Guide

This guide helps you connect the Paradex MCP server to your AI assistant in minutes.

Most setups below require running commands in a **terminal** (Terminal app on macOS, PowerShell on Windows).

## Prerequisites

- Python 3.10 or higher — check with `python3 --version`. If you need to install it, visit [python.org/downloads](https://www.python.org/downloads/).
- `uvx` — a fast Python package runner. Install it by running:

```bash
pip install uv
# or on macOS:
brew install uv
```

---

## Step 1 — Get Your Paradex Private Key

All trading features require a Paradex account private key. Public market data works without one.

1. Go to [Paradex](https://app.paradex.trade) and connect your wallet
2. Navigate to **Settings → API Keys** and create a new key
3. Copy the private key — you'll need it in the steps below

<Note>
**Testnet first?** Use `PARADEX_ENVIRONMENT=testnet` and get test funds from the Paradex testnet faucet.
</Note>

## Claude Desktop

**Fastest path:**

Run this in your terminal to install and auto-configure the Paradex server via Smithery:

```bash
npx -y @smithery/cli install @tradeparadex/mcp-paradex-py --client claude
```

**Manual setup:**

1. Open the Claude Desktop config file in a text editor:
   - **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`

   <Accordion title="How to open this file on macOS">
   **Option 1: The "Go to Folder" Shortcut (Fastest)**

   1. Open Finder.
   2. Press **Command + Shift + G** on your keyboard.
   3. Paste the following path into the box: `~/Library/Application Support/Claude/`
   4. Press **Enter**.
   5. Right-click `claude_desktop_config.json` and select **Open With → TextEdit** (or VS Code).

   ---

   **Option 2: Using the Terminal**

   1. Open Terminal (**Cmd + Space**, type "Terminal").
   2. To open in your default text editor, run:
      ```bash
      open ~/Library/"Application Support"/Claude/claude_desktop_config.json
      ```
   3. Or to edit directly in the terminal:
      ```bash
      nano ~/Library/"Application Support"/Claude/claude_desktop_config.json
      ```

   ---

   **Option 3: Manual Navigation**

   The Library folder is hidden by default.

   1. Open Finder.
   2. Click the **Go** menu at the top of your screen.
   3. Hold down the **Option (⌥)** key — the Library folder will appear in the list.
   4. Click **Library → Application Support → Claude**.
   </Accordion>

   - **Windows**: `%APPDATA%\\Claude\\claude_desktop_config.json`

   <Note>
   **Can't find the file?** Open Claude Desktop → **Settings → Developer → Edit Config**. This opens the file directly. If it doesn't exist yet, create it at the path above.
   </Note>

2. Paste the following (replace `your_private_key_here` with your actual key from Step 1):

```json
{
  "mcpServers": {
    "paradex": {
      "command": "uvx",
      "args": ["mcp-paradex"],
      "env": {
        "PARADEX_ENVIRONMENT": "prod",
        "PARADEX_ACCOUNT_PRIVATE_KEY": "your_private_key_here"
      }
    }
  }
}
```

3. Restart Claude Desktop. It will automatically download and start the MCP server via `uvx` — no separate install needed. You should see a hammer icon (🔨) in the bottom-left of the chat input, indicating MCP tools are loaded.

## ChatGPT Desktop (app)

ChatGPT Desktop app also requires a remote URL. Enable **Developer Mode** in ChatGPT settings first.

### Option A — ngrok tunnel (no cloud account, good for trying it out)

[ngrok](https://ngrok.com) creates a temporary public URL that forwards to your local machine.

1. Install dependencies:
   ```bash
   pip install uv
   brew install ngrok   # macOS; or download from ngrok.com
   ```

2. Start the MCP server locally:
   ```bash
   MCP_TRANSPORT=streamable-http MCP_PORT=8080 \
   PARADEX_ENVIRONMENT=prod \
   PARADEX_ACCOUNT_PRIVATE_KEY=your_private_key_here \
   uvx mcp-paradex
   ```

3. In a second terminal, open a tunnel:
   ```bash
   ngrok http 8080
   ```
   Copy the `https://xxxx.ngrok.io` URL.

4. In ChatGPT Desktop: **Settings → Advanced → Developer Mode** (enable), then **Settings → Connectors → Create**, paste `https://xxxx.ngrok.io/mcp`.

<Note>
**Limitation:** The tunnel URL changes every time you restart ngrok (free tier). Update the connector URL each session.
</Note>

### Option B — Permanent remote endpoint (recommended for daily use)

See the [Configuration & Deployment guide](/agentic-ai-hub/mcp/configuration#remote-http-deployment) for step-by-step instructions (Railway, Render, Fly.io, AWS Lambda, Docker).

Once you have a URL, add it in **ChatGPT → Settings → Connectors → Create**.

### Option C — OpenAI Agents SDK (for developers building custom Python apps)

For custom Python apps (not the ChatGPT interface). No remote hosting needed.

```bash
pip install openai-agents
```

```python
import asyncio, os
from agents import Agent, Runner
from agents.mcp import MCPServerStdio

async def main():
    async with MCPServerStdio(
        name="paradex",
        params={
            "command": "uvx",
            "args": ["mcp-paradex"],
            "env": {
                **os.environ,
                "PARADEX_ENVIRONMENT": "prod",
                "PARADEX_ACCOUNT_PRIVATE_KEY": "your_private_key_here",
            },
        },
    ) as server:
        agent = Agent(
            name="Paradex Trader",
            instructions="You are a trading assistant with access to Paradex.",
            mcp_servers=[server],
        )
        result = await Runner.run(agent, "What are the top markets by volume?")
        print(result.final_output)

asyncio.run(main())
```

## Gemini CLI

Open (or create) `~/.gemini/settings.json` in a text editor and add the `mcpServers` block:

```json
{
  "mcpServers": {
    "paradex": {
      "command": "uvx",
      "args": ["mcp-paradex"],
      "env": {
        "PARADEX_ENVIRONMENT": "prod",
        "PARADEX_ACCOUNT_PRIVATE_KEY": "$PARADEX_ACCOUNT_PRIVATE_KEY"
      }
    }
  }
}
```

The `$VAR_NAME` syntax tells Gemini CLI to read the value from an environment variable instead of storing the key directly in the config file.

Set the variable in your terminal before running Gemini CLI:

```bash
export PARADEX_ACCOUNT_PRIVATE_KEY=your_private_key_here
gemini
```

Manage servers during a session with `/mcp enable paradex` or `/mcp disable paradex`.

## OpenAI Codex CLI

Run this in your terminal to add the Paradex server:

```bash
codex mcp add paradex -- uvx mcp-paradex
```

**Or edit `~/.codex/config.toml` in a text editor:**

```toml
[mcp_servers.paradex]
command = "uvx"
args = ["mcp-paradex"]
env = { PARADEX_ENVIRONMENT = "prod", PARADEX_ACCOUNT_PRIVATE_KEY = "your_private_key_here" }
```

The config is shared between the Codex CLI and the Codex VS Code extension — configure once and both clients pick it up.

## Troubleshooting

**Tools not appearing / MCP not loading**
- Make sure `uvx` is installed: `uvx --version`
- Check that the config file has valid JSON (no trailing commas)
- Restart the client after config changes

**Authentication errors**
- Double-check your private key is correct and starts with `0x`
- Verify `PARADEX_ENVIRONMENT` matches where your account was created (`prod` or `testnet`)

**`uvx: command not found`**
```bash
pip install uv
# or on macOS:
brew install uv
```

**Want to test without a private key first?**
Market data tools work without any credentials. Just omit `PARADEX_ACCOUNT_PRIVATE_KEY` from the config.
