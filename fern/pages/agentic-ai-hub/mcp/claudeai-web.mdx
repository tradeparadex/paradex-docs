# Claude.ai (Web)

Claude.ai web cannot run local programs. Choose a setup option below.

<Tabs>
<Tab title="Remote MCP deployment">

Host your own MCP server on a cloud platform. Market data only.

<Warning>
Remote setups do **not support trading** at the moment — only market data is available.
</Warning>

<Steps>
### Deploy the MCP server

Follow the [Remote Deployment guide](/agentic-ai-hub/mcp/remote-deployment) to deploy the MCP server to a cloud platform. Once you have your endpoint URL, return here.

### Connect your endpoint

1. Open [claude.ai](https://claude.ai) → **Settings → Integrations**.
2. Click **Add MCP Server**.
3. Enter your endpoint URL (e.g. `https://your-domain.example.com/mcp`).

### Verify it's working

Ask Claude:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're connected.
</Steps>

</Tab>
<Tab title="Paradex-hosted MCP">

Market data only. No installation needed, just add a URL.

<Warning>
This setup does **not support trading** at the moment. Only public market data is available.
</Warning>

<Tabs>
<Tab title="Via URL">

<Steps>
### Add the MCP server

1. Open [claude.ai](https://claude.ai) → **Settings → Integrations**.
2. Click **Add MCP Server**.
3. Enter `https://mcp.paradex.trade/mcp`.

### Verify it's working

Ask Claude:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're connected.
</Steps>

</Tab>
<Tab title="Via Smithery">

<Steps>
### Connect through Smithery

1. Visit [smithery.ai](https://smithery.ai) and search for **Paradex**.
2. Select the Paradex MCP server.
3. Choose **Claude** as your client.
4. Follow the guided setup and click **Connect**.

### Verify it's working

Ask Claude:

> "What markets are available on Paradex?"

If you get a list of trading pairs, you're connected.
</Steps>

### Troubleshooting

**MCP server not loading**
- Check that the URL is correct and ends with `/mcp`
- Try removing and re-adding the server in **Settings → Integrations**
- For remote deployments, verify your server is running: `curl https://your-endpoint.example.com/health`
