# CLI

**Repository:** [github.com/tradeparadex/paradex-cli](https://github.com/tradeparadex/paradex-cli)

A crypto DEX AI-native command-line interface for managing Paradex account contracts. Designed for both direct use and AI agent integration, the CLI provides full Paradex API access in a single binary with built-in MCP server support.

What the CLI supports:
- Fetch live and historical market data
- Manage trading accounts and vaults
- Place, monitor, and cancel orders
- Track positions, balances, and funding payments

<Note>
The CLI includes a built-in MCP server for AI agent integration. See [AI Agent Integration](/agentic-ai-hub/cli/ai-integration) for setup instructions and a comparison with the standalone MCP server.
</Note>

## Commands

All commands accept an `--env` flag to specify the target environment (`prod`, `testnet`, or `nightly`).

### Account Management

```bash
# Print account information
paradex_cli print-account-info ACCOUNT_ADDRESS --env ENVIRONMENT

# Add a guardian
paradex_cli add-guardian PUB_KEY --env ENVIRONMENT

# Add a backup guardian
paradex_cli add-guardian-backup PUB_KEY --env ENVIRONMENT

# Change the account signer
paradex_cli change-signer PUB_KEY --env ENVIRONMENT
```

### Transaction Signing & Submission

```bash
# Sign an invoke transaction
paradex_cli sign-invoke-tx FILE_PATH --env ENVIRONMENT

# Submit a signed invoke transaction
paradex_cli submit-invoke-tx TX_FILE_PATH SIG_FILES --env ENVIRONMENT
```

### Emergency Operations

```bash
# Trigger guardian escape mechanism
paradex_cli trigger-escape-guardian --env ENVIRONMENT
```

## Development

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/tradeparadex/paradex_cli.git
cd paradex_cli

# 2. Install dependencies
poetry install

# 3. Run tests
poetry run pytest
```

### Contributing

1. Make your changes
2. Add tests for new features or bug fixes
3. Run tests: `poetry run pytest`
4. Commit and push to your fork:

```bash
git add .
git commit -m "Description of your changes"
git push origin your-branch
```

5. Open a pull request against the `main` branch

### Build & Publish

```bash
# Build
poetry build

# Publish to PyPI (requires configured token)
poetry config pypi-token.pypi <your-token>
poetry publish --build
```
